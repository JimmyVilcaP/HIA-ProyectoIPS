Metodología Scrum y DevOps

Declaración explícita de herramientas de gestión ágil y automatizaciones implementadas en el ciclo de vida de HIA.

Herramientas Scrum utilizadas

  • Jira (Gobernanza del Sprint Backlog)
  • GitHub Projects (Tablero Kanban de Evidencias)
  • GitHub Issues (Trazabilidad de requerimientos técnicos)

Herramientas DevOps utilizadas

  • GitHub Actions (Servicio de Integración Continua)
  • GitHub Pages (Alojamiento estático de documentación)
  • Streamlit Cloud (Entorno de despliegue productivo de la app)
  • GitHub Repository (Gestor de código fuente centralizado)
  • Git Branching Strategy (Implementación de ramas alternativas: main/develop)

Flujo de Integración Continua (CI)

El proyecto HIA implementa un proceso de Integración Continua (CI) mediante GitHub Actions. Cada actualización enviada a las ramas principales del repositorio activa automáticamente una pipeline encargada de instalar dependencias, validar la sintaxis del código fuente y ejecutar pruebas automatizadas desarrolladas con Pytest. Este proceso permite detectar errores de manera temprana, garantizar la estabilidad del sistema y mantener una base de código confiable antes de su despliegue en Streamlit Cloud, siguiendo buenas prácticas DevOps aplicadas al desarrollo de software.
