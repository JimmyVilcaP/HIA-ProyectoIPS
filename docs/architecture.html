Arquitectura del Proyecto

Diseñada para ofrecer una arquitectura modular que facilite el mantenimiento, la escalabilidad y la integración con servicios de inteligencia artificial.

Diagrama Conceptual del Sistema

Estructura por Capas Reales

Capa de Presentación

Desarrollada sobre Streamlit, encargada de renderizar Auth Pages, Sidebar de control, Analysis Form y la Chat Interface.

Capa de Lógica de Negocio

Orquestada mediante LangChain, compone el Analysis Agent, Chat Agent, Model Manager, PDF Extractor y validadores de estructura.

Capa de Datos

Utiliza Supabase (PostgreSQL) para Chat Sessions y Chat Messages junto a almacenamiento vectorial local en FAISS con HuggingFace Embeddings.

Flujo de Integración y Datos

El sistema procesa los archivos PDF médicos, extrayendo el texto plano y segmentándolo para generar vectores relacionales mediante HuggingFace Embeddings que se alojan temporalmente en FAISS. Las consultas conversacionales se resuelven llamando a la API de Groq. Las sesiones de usuario, análisis generados y mensajes del chat se almacenan en Supabase para permitir la persistencia y recuperación de información entre sesiones.
